## 1. 设计语言总览

延续参考原型图确定的方向：**轻量像素风（Lightweight Pixel / 8-Bit）**。

| 关键词 | 含义 |
|---|---|
| 像素插画 | 角色、奖牌、器材、装饰均为像素美术（已切图，见 §4）。 |
| 硬边扁平 | 无渐变、无柔和阴影；卡片/按钮用 **2px 深色描边 + 硬实心偏移阴影** 制造「方块凸起」感。 |
| 清爽浅色底 | 浅灰/薄荷底，绿色主色，氛围轻松、正反馈强。 |
| 高信息密度 | 训练页优先信息密度与记录效率，插画克制（对齐 PRD §3.3 / §6.3）。 |
| 游戏化 | 连胜火焰、奖牌、进度条等游戏化元素强化坚持动机。 |

**风格分工原则（重要，决定每个元素怎么做）：**

- **结构性控件用 CSS 画**（无需切图，灵活自适应）：按钮外形、卡片描边/阴影、分段控件、标签 chip、进度条填充、勾选/状态方块、每周进度方块、热力图格子。
- **插画性元素用 PNG 切图**（已备齐）：角色、奖牌/奖杯、器材图标、功能图标（麦克风/火焰/日历/编辑/箭头/星/喇叭/加减）、装饰（云/草地/爱心）。

> 即「**CSS 负责形状与排版，PNG 负责美术**」。这样既保证像素质感，又不被固定尺寸图片限制布局。

## 2. 设计 Token

> 落地于 `miniprogram/styles/tokens.wxss`。现有 token 已大部分可复用；本次像素化**新增**描边色、像素阴影、像素字体，并收紧圆角。改全局换肤只改这里。

### 2.1 颜色

| Token | 值 | 用途 |
|---|---|---|
| `--c-primary` | `#3DD16B` | 主绿：完成、训练中、正向、主按钮、选中态。 |
| `--c-primary-strong` | `#0F8A43` | 深绿：描边内的高光/按下态、进度条描边。 |
| `--c-primary-soft` | `#E9F7EE` | 浅绿底：数据卡、明细卡背景。 |
| `--bg-mint` | `#EEF6EF` | 薄荷面板底（连胜卡等）。 |
| `--c-blue` | `#4E8DF5` | 数据/进行中状态（蓝色徽标、趋势）。 |
| `--c-amber` | `#F5BE4B` | 成就/警示性操作（如「完成/结束」小按钮、时长卡）。 |
| `--c-purple` | `#8C7AE6` | 辅助点缀（部分奖牌）。 |
| `--c-pink` | `#F5285E` | 仅删除/风险（对齐 PRD §6.3）。 |
| `--bg-page` | `#F6F8F8` | 页面底色。 |
| `--surface` | `#FFFFFF` | 卡片底色。 |
| `--text-1/2/3` | `#1A1D1F` / `#6E7378` / `#A8ACB0` | 主/次/弱文字（弱文字＝Tab 未选中色）。 |
| **`--c-ink`** 🆕 | `#2B2B2B` | **像素描边色**（所有方块控件的 2px 描边）。 |

> 像素风**弃用**原 `--bg-gradient` 渐变背景（与硬边像素冲突），页面统一纯色 `--bg-page` + 角落像素云/草地装饰（见 §6.1）。

### 2.2 像素描边与阴影 🆕

像素风的「凸起」靠描边 + 硬偏移阴影，**不要用模糊阴影**：

```css
/* 方块控件通用 */
--px-border: 4rpx solid var(--c-ink);     /* 屏幕上≈2px 描边 */
--px-radius: 12rpx;                        /* 像素圆角，收紧，不要 32rpx 大圆角 */
--px-shadow: 6rpx 6rpx 0 0 var(--c-ink);   /* 硬实心偏移阴影（无 blur） */
--px-shadow-press: 2rpx 2rpx 0 0 var(--c-ink); /* 按下：阴影变小+位移，模拟下沉 */
```

> 旧 token 中的 `--shadow-card`（模糊阴影）、`--r-card: 32rpx`（大圆角）在像素风页面**不再使用**；保留以兼容未改造页面，改造完可删。

### 2.3 字体

| 场景 | 字体 | 说明 |
|---|---|---|
| 数字 / 英文 / 大标题数字 | **像素字体**（Press Start 2P 或 Zpix） | 计时器、统计大数、Lv./EXP、PR 重量等「signature 数字」。 |
| 中文正文 | 系统字体 或 **Zpix 子集** | 见 §3 字体策略——主包够塞子集，但首版可先系统字体快速跑通。 |

字号沿用「**标题粗重、数字更大、正文克制**」（PRD §6.3）：

| 角色 | 字号(rpx) | 字重 |
|---|---|---|
| 页面大标题 | 40 | 800 |
| 区块标题 | 32–34 | 600 |
| signature 大数字（计时/统计） | 64–80 | 800（像素体） |
| 卡片标题 | 30 | 700 |
| 正文 | 28 | 400–500 |
| 辅助/单位 | 22–24 | 400 |

### 2.4 间距与圆角

沿用现有 8rpx 网格：`--sp-1..6 = 8/16/24/32/48/64rpx`。圆角统一改用 `--px-radius: 12rpx`（卡片）/ `--r-pill` 仅用于个别胶囊。

### 2.5 像素渲染

切图为像素美术，**避免放大超过原生尺寸导致糊边**；需要时在 `image` 上加 `image-rendering: pixelated`（部分基础库支持）。优先**按整数倍缩小**展示。

## 3. 字体策略

主包当前 1.4M，距 2MB 上限有 ~600KB 余量，足够塞中文像素字体子集。落地分两步：

1. **首版（先跑通）**：数字/英文用 `Press Start 2P`（几十 KB，打包）；中文用系统字体。不阻塞开发。
2. **精修（统一像素感）**：中文上 **Zpix 最像素**，二选一：
   - **子集化打包**（推荐）：用 `fontmin` 抽取项目实际用到的汉字（动作名/界面词，约数百字），压到 ~50–150KB，直接打包、零延迟。
   - **`wx.loadFontFace` 远程加载**：完整 Zpix 放 CDN，运行时加载、不占包体积，代价是首屏可能闪一下默认字体。

> 字体文件不入 git 大包前先确认子集体积；接入样例开发时再补。

## 6. 页面规格

四个 Tab 对齐 PRD §3.1。下列布局按**参考原型图**逐屏拆解，并标注与 PRD 的差异。

### 6.1 通用框架

| 元素 | 规范 |
|---|---|
| 状态栏 | 自定义导航，顶部留 `statusBarHeight` 高度（各页已有）。 |
| 页面底 | 纯 `--bg-page`；四角点缀 `decoration/cloud_*`（顶部）、`landscape_*`（底部山/草），低饱和、不抢内容。 |
| TabBar | 自定义 tabbar（`custom: true`）。4 项图标用 `nav/*`，选中态绿、未选灰；选中文字 `--c-primary`。**命名（决策 #3）：首页 / 训练 / 数据 / 我的**（沿用 PRD 与现有 `custom-tab-bar` 代码，不用参考图的「记录/统计」）。 |
| 滚动 | 列表/长页用 `scroll-view`；底部留 `safe-bottom`。 |
| 主按钮 | 「按住说话」全宽主绿按钮常驻于首页与训练页底部安全区上方。 |

### 6.2 首页（Dashboard）

**布局（自上而下，参考图①）：**
1. **标题行**：左为问候语 `下午好，Alex` + 日期（决策 #4 并入此处）+ 右侧方块「+」按钮（快捷开始/记录入口）。品牌名 `开练 KaiLift` 走导航栏标题。
2. **本月训练天数卡** `streak-banner`：薄荷底方块卡；左 `overhead_press` 角色，右「本月训练 / 已训练 X 天」+ `fire` 图标 + 「继续保持 →」按钮（**保留火焰/卡片视觉**，文案改为本月已训练天数，见决策 #1）。
3. **本周轨迹** `week-tracker`：区块标题 + 7 方块（一~五完成绿勾，六/日未完成）。
4. **上次训练** `card`：`equipment/bench` 缩略 + `胸部 · 杠铃卧推` + `2024/05/18 · 75 min` + `arrow_right`，点击进 `session-detail`。
5. **本周概览**：3 张 `stat-card`——训练次数(次/绿) `calendar`、总训练量(kg/蓝) `barbell`、训练时长(小时/琥珀) clock。
6. **主按钮** `voice-button`：`按住说话`。

**已定决策（详见 §9）：**
- **#1** 连胜卡文案＝「本月训练 · 已训练 X 天」（取代 21 天计划），**不写「连续打卡」**（保留火焰/卡片视觉）。
- **#4** 顶部问候语「下午好，Alex」**并入标题行**（标题行左为问候+日期，或品牌名与问候二选一，开发时取其一即可）。
- **#5** 区块命名以 PRD 为准：「**本周轨迹**」「**本周概览**」（非「每周/月度」）。

### 6.3 训练页（Live Logging）

**这是高频核心工具页**，PRD §3.3 要求优先记录效率而非插画。含状态机。

**「训练中」布局（参考图③）：**
1. **顶栏**：左 `完成` 小按钮（琥珀，`small_done` 风格）+ 居中标题 `记录中`。
2. **角色探头**：计时卡上沿露出 `male_face_*`（氛围，可省）。
3. **计时卡** `timer`：`✦ 已用时间 ✦` + 大计时 `24:18`（像素体）+ 虚线分隔 + 三栏：组数 / 动作数 / 总负荷(kg)。
4. **当前动作栏**：`当前动作 › 杠铃卧推` + 绿点（对齐 PRD「当前动作上下文」`current_workout_exercise_id`）。
5. **动作卡列表** `exercise-card`：每卡含器材图标 + 动作名 + `组×次@重量` + 状态徽标（绿勾=完成 / 蓝钟=进行中）；其下组明细行：`第N组 · 100kg × 5`，行首状态点（绿勾=已记/蓝播放=当前组）。
6. **底部**：全宽 `按住说话` 主按钮 + 提示 `例如：卧推100公斤5次` + 右侧 `···` 更多（=「+ 手动加组」/编辑入口，对齐 PRD 常驻手动加组）。

**状态机（对齐 PRD §3.3）：**
| 状态 | 表现 |
|---|---|
| 未开始 | 空态插画（`standing_front`）+「开始自由训练」按钮。 |
| 训练中 | 上述布局。 |
| 语音识别中 | 麦克风动效 +「正在听…」+ 可取消。 |
| 解析清晰自动保存 | 短提示「已记录 N 组」+ 新组高亮 + 撤销。 |
| 需要确认 | 弹解析确认卡（展示拆出的多组，可确认/编辑/放弃）。 |
| 语音失败/弱网 | toast +「改用手动」打开手动加组表单。 |

> 录音交互：toggle（点开始/点结束）+ 静音 1.5s 自停 + 20s 上限，全程可取消（PRD §3.3）。

### 6.4 数据页（Stats）

**布局（参考图②）：**

## 10. 实现注意（小程序）

- 单位：尺寸用 `rpx`；像素描边 `4rpx`≈2px、阴影 `6rpx`，整数避免半像素糊。
- 像素图缩放：优先整数倍缩小，必要时 `image-rendering: pixelated`。
- 主包体积：当前 1.4M，余量 ~600KB；新增切图前留意 §3 字体预算。
- token 改造：像素化新增 `--c-ink`/`--px-*` 到 `tokens.wxss`；旧 `--shadow-card`/`--r-card` 改造完页面后可清。
- 组件复用：优先改造 `components/*` 既有组件，勿另起炉灶。
